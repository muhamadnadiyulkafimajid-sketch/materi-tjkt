💻 PERINTAH TEKS JARINGAN DI COMMAND PROMPT

CMD (Command Prompt) adalah instruksi berbasis teks untuk ping (mengecek konektivitas), IP config (jalur data), netstat (statistik koneksi), dan NS lookup (info DNS) yang semuanya digunakan untuk pemecahan masalah dan konfigurasi jaringan di Windows.

📌 PERINTAH JARINGAN DASAR DAN FUNGSINYA

IP config: menampilkan konfigurasi IP (alamat IP, subnet mask, gateway default)
ipconfig/all: menampilkan konfirmasi detail lengkap
ipconfig /flushdns: membersihkan cache DNS
IP config /release & renew: melepas dan mendapat IP baru
ping [alamat]: menguji konektivitas ke host lain (misal: ping google.com)
tracert [alamat]: melacak jalur paket data ke tujuan (misal: tracert Google.com)
netstat: menampilkan koneksi jaringan aktif, tabel routing dan statistik
nslookup [nama_domain]: memeriksa informasi DNS untuk nama domain (misal: nslookup google.com)
route: menampilkan atau memanipulasi tabel routing IP
hostname: menampilkan nama komputer anda

🔧 Cara Membuka CMD

Tekan tombol Windows + R, ketik CMD, lalu tekan enter atau klik start, ketik CMD, dan pilih untuk membukanya.

📋 Contoh Penggunaan

✔ Cek koneksi ke Google: ketik ping google.com lalu enter. jika berhasil, akan muncul respons waktu (Time) dan byte
✔ Lihat alamat IP Anda: ketik ipconfig lalu enter, cari "ipv4 address"
✔ Cari tahu alamat IP dari domain: ketik nslookup www.facebook.com lalu enter

🌐 IP ADDRESS

IP address => merupakan alamat dalam jaringan yang terhubung antara client 1 dan 2 dan seterusnya. Di dalam IP address client terdapat gateway.
Apa itu gateway? => lalu lintas atau jalan di dalam IP address. Terdapat juga subnet mask.
Subnet mask => seolah kendaraan.
IP address adalah nomor penumpang, gateway jalannya atau lalu lintas (kabel dan nirkabel).
Alamat adalah berupa angka yang dikatakan dalam klien.

Kelas IP Address:

• Kelas A → 1.0.0.0 sampai 126.0.0.0

• Kelas B → 128.0.0.0 sampai 191.255.0.0

• Kelas C → 192.0.0.0 sampai 223.255.255.0

• Kelas D → 224.0.0.0 sampai 239.255.255.255

• Kelas E → 240.0.0.0 sampai 255.255.255.255

🖥️ APLIKASI VIRTUAL

🔷 VirtualBox

Cara kerja: VirtualBox bekerja dengan membuat virtual machine (VM) yang meniru perangkat keras komputer. Setiap VM bisa menjalankan sistem operasi berbeda di atas host OS.

Fungsi dan manfaat:
- Menjalankan beberapa OS sekaligus (Windows, Linux, macOS)
- Testing software tanpa risiko ke sistem utama
- Simulasi jaringan virtual
- Isolasi lingkungan kerja

Kegunaan: Cocok untuk developer, pelajar, dan IT yang butuh uji coba sistem atau aplikasi.

Konfigurasi:
1. Install VirtualBox di host OS
2. Buat VM (virtual machine) lalu tentukan RAM, CPU, dan storage
3. Pasang ISO OS yang diinginkan
4. Atur jaringan (NAT, BRIDGE, HOST-ONLY)

🔷 VMware

Cara kerja: VMware menggunakan teknologi virtualisasi untuk membagi sumber daya server fisik menjadi beberapa VM, setiap VM berjalan seolah-olah memiliki hardware sendiri.

Fungsi dan manfaat:
- Menjalankan banyak OS dalam satu perangkat
- Mendukung skala besar untuk data center dan cloud
- Mempermudah manajemen server dan aplikasi
- Efisiensi biaya dan fleksibilitas tinggi

Kegunaan: Digunakan perusahaan untuk cloud computing, server virtualization, dan pengelolaan infrastruktur IT.

Konfigurasi:
1. Install VMware Workstation/ESXI
2. Buat VM lalu tentukan spesifikasi hardware virtual
3. Pasang OS via ISO
4. Atur integrasi dengan vSphere/vCenter untuk manajemen skala besar

🔷 Proxmox

Cara kerja: Proxmox adalah platform open-source berbasis web untuk mengelola VM dan container. Mendukung KVM (virtualisasi penuh) dan LXC (container).

Fungsi dan manfaat:
- Open-source dan gratis
- Mendukung clustering dan high availability
- Firewall bawaan dan integrasi dengan storage

Kegunaan: Ideal untuk perusahaan kecil hingga data center besar yang membutuhkan solusi hemat biaya tapi powerful.

Konfigurasi:
1. Install Proxmox VE di server fisik
2. Akses via web interface
3. Buat VM atau container dan tentukan resource
4. Atur storage, jaringan, dan clustering jika diperlukan
